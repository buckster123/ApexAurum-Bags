### Dividend Distribution

Top 100 holders receive automatic dividends from Bags.fm:

```python
# How dividends work (Bags.fm automatic)
# Every 24 hours, if unclaimed fees >= 10 SOL:

total_fees = get_unclaimed_fees()  # e.g., 50 SOL
top_100 = get_top_100_holders()

for holder in top_100:
    share = holder.balance / sum(h.balance for h in top_100)
    payout = total_fees * share
    # Auto-deposited to holder's wallet
```

### Database Schema

```sql
-- Wallet authentication
CREATE TABLE wallets (
    id UUID PRIMARY KEY,
    address VARCHAR(44) UNIQUE NOT NULL,
    user_id UUID REFERENCES users(id),
    created_at TIMESTAMP DEFAULT NOW(),
    last_verified TIMESTAMP
);

-- Token balance cache
CREATE TABLE token_balances (
    wallet_address VARCHAR(44) PRIMARY KEY,
    balance BIGINT NOT NULL,
    tier VARCHAR(20) NOT NULL,
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Governance proposals
CREATE TABLE proposals (
    id UUID PRIMARY KEY,
    proposer_wallet VARCHAR(44) NOT NULL,
    title VARCHAR(200) NOT NULL,
    description TEXT NOT NULL,
    proposal_type VARCHAR(50) NOT NULL,
    options JSONB NOT NULL,
    votes JSONB DEFAULT '{}',
    status VARCHAR(20) DEFAULT 'active',
    created_at TIMESTAMP DEFAULT NOW(),
    ends_at TIMESTAMP NOT NULL,
    executed_at TIMESTAMP
);

-- Votes
CREATE TABLE votes (
    id UUID PRIMARY KEY,
    proposal_id UUID REFERENCES proposals(id),
    wallet_address VARCHAR(44) NOT NULL,
    option VARCHAR(50) NOT NULL,
    weight BIGINT NOT NULL,
    signature VARCHAR(128) NOT NULL,
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(proposal_id, wallet_address)
);

-- Dividend tracking
CREATE TABLE dividend_claims (
    id UUID PRIMARY KEY,
    wallet_address VARCHAR(44) NOT NULL,
    amount_sol DECIMAL(20, 9) NOT NULL,
    tx_signature VARCHAR(128) NOT NULL,
    claimed_at TIMESTAMP DEFAULT NOW()
);
```

## Risk Considerations

### Technical Risks

| Risk | Mitigation |
|------|------------|
| Smart contract bugs | Use audited Bags.fm/Meteora contracts |
| API rate limits | Implement caching, fallbacks |
| Wallet hacks | Never store private keys, use signatures |
| RPC downtime | Multiple RPC providers (Helius, Quicknode) |

### Market Risks

| Risk | Mitigation |
|------|------------|
| Low trading volume | Focus on utility, not speculation |
| Token price crash | Strong fundamentals, buybacks |
| Competitor tokens | Unique AI + token integration |
| Regulatory changes | Legal review, geo-restrictions if needed |

### Operational Risks

| Risk | Mitigation |
|------|------------|
| Team key loss | Multi-sig wallets, backup procedures |
| Governance attacks | Quorum requirements, time delays |
| Community toxicity | Clear guidelines, moderation |

### Legal Considerations

- [ ] Consult crypto-friendly legal counsel
- [ ] Review securities laws in key jurisdictions
- [ ] Implement geo-blocking if needed (US?)
- [ ] Clear terms of service
- [ ] No investment promises/guarantees
